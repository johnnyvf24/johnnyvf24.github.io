HelloChess

My final year in college, I worked on a side project which I named HelloChess, a chess website with 4 player chess. The initial version of HelloChess was written in PHP using the Codeigniter framework, but I still relied on NodeJS for real-time moves. After a while I decided to revamp the project, I rewrote most of the website to run entirely on NodeJS, Socket.io, and React with Redux. After a while I met another enthusiastic chess-player/programmer named Joe who added S-Chess, Crazyhouse960 and FullHouse (a new variant). Given how much Joe helped me, I decided to make him co-owner of the website. Currently we both still contribute to the website whenever we have spare time. The website can be found here and the code on Github

MESH Web Application

From early 2016 till my graduation in May 2017 I worked for Dr. William Yeoh on a National Science Foundation funded project. The project, which we named MESH, (Mult-agent Energy-efficient Smart Homes) was a distributed Smart Home project with the aim of reducing peak power consumption usage (e.g. the time of day when energy costs the most). The project was great to work for, unfortunately, we met several hickups and one of them required a re-write of our code-base. After I realized most of my code would need to be re-written I decided to re-write my PHP/Codeigniter application into NodeJS and Express. Long story short my project was extremely dependent upon changes to each house agent controlled in Java and constant changes to the database schema resulted in medial progress. Ultimately a basic version of the application was finished and I am really happy with the resulting application that used TypeScript, React, Redux, and NodeJS. If you want to read more about the project please visit this website (which I also built).

  • Client: MESH
  • Date: Jan 2016 - May 2017
  • Topic: Web Development

Tetra Chess AI

I had finally finished the initial version of HelloChess! Unfortunately, (also fortunately) my curiosity got the best of me. My mind dwelled on how cool it would be to build a 4 player chess AI. Thoughts are expensive, perhaps not in money but definitly in effort, time, and grey hair. I spent a solid 3-4 weeks programming the initial version of Tetra in September 2016. My first version was written in JavaScript, but I should have known better. JavaScript is not the most performant language and I would need alot of power to calculate all those positions (also to compensate for my less than ideal code). Ultimately I rewrote the engine in C. After several iterations, I had soemthing to show. The engine was capped at a depth of about 6 ply (1.5 turn/board rotations) before the engine would take exceedingly long, but the engine was still strong enough to beat most human players. Furthermore, having 3 engines of equal thinking makes the game much harder!

  • Date: September 2016
  • Topic: Artificial Intelligence

Kalah Agent

During the Spring of 2016, my junior year at university, I was taking an Artificial Intelligence class. At the end of the semester my teacher had a class competition to see who could program the best Kalah agent. The winner of the class would get extra points! Seeing as I had slacked off a little too much, I decided to enter the competition. Long story short, GMKalah (what I called my agent) won the competition. It was quite nervewracking to see my AI down almost 30 beans in one round! Luckily, the AI pulled through to win. The Agent used very traditional techniques to win the competition. The most complicated part was alpha-beta search with iterative deepening. My attempts at parallelizing the agent were futile as there just wasn't enough time for a more elaborative AI. What was my secret then? A very sophisticated evaluation function that would allow me to reach depth 18 within reasonable time. In fact, I was unable to find another agent on the internet that could beat GMKalah (ignoring the fact that someone has solved the game now).
